About this website thing

As you may have guessed, this is just another one of those sites (not very profound, not that they intend to in any way). QRGames allows you to code small JavaScript games using an internal "framework" (using a generous nomenclature) and to share them using QR codes. These would contain all the game's code by means of minification and compression, so no server is necessary to either store the code nor to run it.

By using this website's internal JavaScript editor (it's just Ace), you can code and test them easily before you share them. Be aware that games shared with this technology might contain malicious code, but this is completely not my responsibility. Do not blame me if you execute some bad code on this website (pls no sue).

Does it serve any purpose?

Quick answer: not really.

Long answer: do any of these silly experiments serve any purpose, really? I just thought it would be funny when I found out about 1KJS, this guy that put a game in a QR code and some incredible engineering feats such as David Horne's 1K ZX Chess. This website's sole purpose is to make people feel accomplished as they have their own QR containing a game they made. It also enables people to share small games with only a photograph, which is crazy.

Let's be real for a second, this is just a fully client-side web and I do not intend to keep updating it very often. This has been a project for like a week or so, and now it is functional and over (unless it gains some media attention, but I seriously doubt it).

Have fun coding silly games!
