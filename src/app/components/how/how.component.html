The basics

This web's execution environment is based on HTML's canvas element. The code runner automatically instantiates a canvas and wraps your code inside a lambda expression with the 2D drawing context as its only parameter. This parameter will be called c and you have to know how to work with it, since it is the only thing that you will use to interact with the game.

As you may have guessed, this parameter is modified in order to to allow the usage of shorter code. The modifications are the following (access them by using c.whatever):

Name Type Description
h Attribute Canvas height
w Attribute Canvas width
t Attribute Current tick (increases by 1 on every frame)
e Attribute Event container. Click events and so on are stored in here
l(x1, y1, x2, y2, w=1, c='#000000') Function Draw line of width w and color c from (x1, y1) to (x2, y2)
c(x, y, r, w=1, c='#000000') Function Draw circle of radius r, width w and color c at position (x, y)
...

Aaaand I can't be bothered to fill out the entire table, I think you get the point. The full list is on GitHub (you can check the runner.component.ts file, ngAfterViewInit function).

How to run games

Games, as they usually do, have a main loop. This is a fancy word for a snippet of code that is executed several times in order to refresh the screen. This snippet is, in your case, the function you wrote in the editor. This function will be run every frame (60 times per second) and will draw or delete what you tell it to.

You can see some examples in the examples folder in GitHub.
